# Roadmap MVP (2 semanas)

## Semana 1 - Diseno y especificacion
### Dia 1
- Cerrar PRD, modelo matematico y contratos JSON.
- Definir estructura final del monorepo.

### Dia 2
- Definir datasets de prueba base y expected outputs.
- Especificar criterios de validacion de input.

### Dia 3
- Redactar ADRs tecnicas (CLI vs libreria, framework API, stack web).
- Congelar version `v1` del contrato API.

### Dia 4-5
- Setup base de monorepo y CI minima.
- Skeleton de apps y packages sin logica de negocio.

## Semana 2 - Implementacion MVP
### Dia 6-7
- Implementar motor C++ con Edmonds-Karp.
- Pruebas unitarias del motor.

### Dia 8
- Integrar API con ejecucion del motor CLI.
- Manejo de errores y validaciones de entrada.

### Dia 9
- Implementar UI web minima (input + resultado).
- Export JSON/CSV del resultado.

### Dia 10
- Pruebas end-to-end y hardening.
- Preparar demo de portafolio (capturas, gif, script de presentacion).

## Entregables finales
- Demo funcional end-to-end.
- Documentacion tecnica completa.
- Repositorio listo para revision en portafolio.
